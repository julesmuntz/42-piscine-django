In Django, the template engine is the part of the framework that handles rendering HTML pages by combining templates with data from the views. Here’s how it works in Django:

  1. Template File:

    • In Django, templates are stored in a templates/ directory. These files contain static HTML mixed with special Django Template Language (DTL) syntax for dynamic content.
    • Example template (stored as home.html): 
      <h1>{% verbatim %}{{ title }}{% endverbatim %}</<h1>
<p>{% verbatim %}Welcome to {{ site_name }}!{% endverbatim %}</p>
Here, {{ title }} and {{ site_name }} are placeholders for dynamic content that will be inserted.

  • View Function:

    • The view function (in views.py) processes the incoming request and passes data (as a dictionary) to the template. It tells Django to render a specific template and provides the values for placeholders in that template.
    • Example view function: 
      from django.shortcuts import render

def home(request):
    context = {
        'title': 'Home Page',
        'site_name': 'MyWebsite'
    }
    return render(request, 'home.html', context)
    • In this example, the context dictionary contains the values Home Page and MyWebsite for the title and site_name placeholders.

  • Rendering the Template:

    • Django’s template engine takes the home.html template and the context dictionary from the view function.
    • It replaces the placeholders {{ title }} and {{ site_name }} with the actual values from the context: 'Home Page' and 'MyWebsite'.

  • Returning the Response:

    • After rendering the template with the dynamic data, Django returns the complete HTML page as the HTTP response.
    • The client (browser) receives this response and displays the fully rendered static page: 
      <h1>Home Page</h1>
<p>Welcome to MyWebsite!</p>

    • In Summary:

    • Template file: Contains HTML with placeholders ({{ ... }}) for dynamic content.
    • View function: Gathers data and sends it to the template.
    • Django template engine: Combines the template and the data, replacing the placeholders with actual values.
    • Final output: A complete HTML page is sent to the user.

    Django's template engine is what transforms the templates with data into fully rendered web pages!
